Section 6 — Error Handling, Packaging & Testing

Every failure mode has a clear user-facing response. Packaging produces signed installers. Testing strategy keeps CI fast.

⚠️ Error handling — every failure has a clear user message
Oracle Instant Client missing
Detected on startup before any connection attempt. Full-screen guide shown with download link, path instructions per OS. App remains usable for XSD/PG.
Connection timeout
10s timeout with spinner. Error distinguishes: host not reachable vs host reachable but auth failed vs DB not found. Each has a specific fix hint.
DB permissions insufficient
User lacks SELECT on information_schema / ALL_TABLES. Show which tables failed with the exact privilege required. Partial introspection still proceeds for accessible tables.
Oracle TNS resolution failure
tnsnames.ora not found or alias not in it. Show the path where Oracle looks for it, suggest switching to Easy Connect string instead.
Stats query timeout
AVG/COUNT queries on huge tables time out after 5s. Silent fallback to static estimation. MetricsBar shows source: "static" indicator as usual — no error shown.
Connection lost mid-session
Detected when next IPC call fails. Toast notification: "Lost connection to Production PG — Reconnect". Schema and selection state preserved. Reconnect re-runs just the connect step, not the full wizard.
Large schema introspection
Databases with 500+ tables: table list loaded lazily, columns fetched on demand (not all upfront). Progress bar shown. Stats use Oracle SAMPLE / PG TABLESAMPLE for fast approximations.
Auto-update failure
Update check silently fails if offline — no crash, no intrusive dialog. Update available notification is a dismissable toast only, never blocking.

🧪 Testing strategy
Existing 568 Vitest tests — unchanged
All run against the web build. The renderer React code has no Electron dependency — tests continue to work exactly as today.
Unit tests — schema mapping
Pure functions: DB type → XSD type mapping, FK cycle detection, self-ref depth capping, column name sanitisation, composite FK auto-flattening. All Vitest, no DB needed.
Integration tests — PostgreSQL
Docker Postgres container in CI. Tests cover: connect, getTables, getColumns, getForeignKeys, buildSchemaTree, fetchSampleStats. Fast — container spins up in ~5s.
Integration tests — Oracle
Skipped in CI (Oracle Docker image is 2GB+, requires license). Run locally against a real Oracle instance. CI tests the Oracle code path with mocked responses only.
IPC handler tests
Main process IPC handlers tested in isolation (Node.js, no Electron binary needed). DB driver mocked. Covers all error paths — timeout, auth failure, permission error, etc.
E2E — Playwright (Electron driver)
Playwright can drive Electron apps via _electron API. Smoke tests: app launches, XSD upload works, DB tab visible, connection dialog opens. Runs against the packaged app in CI.
