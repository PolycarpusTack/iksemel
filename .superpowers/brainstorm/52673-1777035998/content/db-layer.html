Section 3 — Database Layer & Edge Cases

PostgreSQL is straightforward. Oracle has significant edge cases that need explicit handling.

🐘 PostgreSQL — pg driver (pure JS, no native deps)
Connection inputs
Host, port (default 5432), database
Username + password (keychain-stored)
SSL mode: disable / require / verify-full
Optional: custom CA certificate path
Schema filter (default: public)
Edge cases
Self-signed SSL certs common in enterprise — rejectUnauthorized: false toggle needed
Multiple schemas per DB — user picks which schema(s) to introspect
Connection pooling via pg.Pool — idle timeout 30s, max 3 connections
🔶 Oracle — oracledb driver (native bindings required)
Connection inputs
Easy Connect string: host:port/service
TNS alias (reads local tnsnames.ora)
Username + password (keychain-stored)
Wallet directory path (for TLS/mTLS)
Schema/owner filter
⚠️ Critical edge cases
Oracle Instant Client must be installed. App detects on startup and shows a clear install guide if missing — not a silent crash.
Identifiers stored UPPERCASE unless quoted — normalise to lowercase in SchemaNode names
Oracle DATE includes time component — map to xs:dateTime, not xs:date
NUMBER with no precision = float — map to xs:decimal
Exclude system schemas: SYS, SYSTEM, MDSYS, CTXSYS, XDB, APEX_* etc.
Synonyms — resolve to base table for introspection
CLOB/BLOB columns — excluded from schema tree (not serialisable to XML)
⚡ Schema mapping edge cases — both engines
Circular FKs (A→B, B→A) — detect cycles during tree build, break at the deepest edge. Show warning in UI.
Self-referential FKs (e.g. category → parent_category_id) — render as optional recursive child element, max depth capped at 3.
Composite FK keys — FK relationships spanning multiple columns shown as flat, not nested (nesting multi-key FKs produces ambiguous XML).
Very wide tables (500+ columns) — introspect in batches, show progress indicator. Columns still loaded fully.
NULL / NOT NULL → minOccurs="0" vs minOccurs="1" in schema nodes.
Reserved XML names (e.g. column named "xml", "xsl") — prefix with table name to avoid invalid element names.
Views — included with a "view" badge. No FK introspection for views (no FK metadata). Flat columns only.
DB type → XSD type — full mapping table (VARCHAR→xs:string, INT→xs:integer, DATE→xs:date, TIMESTAMP→xs:dateTime, NUMERIC→xs:decimal, BOOLEAN→xs:boolean, etc.)
🔑 Credential storage
Profile file ~/.xfeb/profiles.json
Stores host, port, database, username, engine, label, isFavourite — everything except passwords
OS Keychain (keytar)
Passwords only. Key: xfeb/{profileId}. Windows Credential Manager on Windows, Keychain on macOS.
