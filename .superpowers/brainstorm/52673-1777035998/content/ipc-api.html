Section 2 — IPC API (window.electronAPI)

The full typed surface the renderer can call. Grouped by concern — each group maps to one IPC handler module in main.

🔌 connections — profile management & live sessions
// test without saving
testConnection(profile) → TestResult
// open a pooled connection
connect(profileId) → ConnectionResult
disconnect(connectionId) → void
listProfiles() → ConnectionProfile[]
saveProfile(profile) → ConnectionProfile
deleteProfile(profileId) → void
setFavourite(profileId, bool) → void
🗂 schema — introspect DB structure
// tables + views
getTables(connectionId) → TableInfo[]
getColumns(connectionId, tableId)
→ ColumnInfo[]
getForeignKeys(connectionId)
→ ForeignKeyRelation[]
// builds SchemaNode[] for the app
buildSchemaTree(connectionId, options)
→ SchemaNode[]
buildSchemaTree options: which FK relationships to nest (the user-choice from Section 5), whether to include views, schema/owner filter for Oracle.
📊 stats — feeds the size estimator (replaces synthetic estimates)
fetchSampleStats(connectionId, columns)
→ FieldSampleData[]
fetchCardinality(connectionId, tableId)
→ CardinalityStats
getRowCount(connectionId, tableId)
→ number
These map directly onto the existing DataProvider interface — a new DatabaseProvider class implements it using these IPC calls. The estimation hook needs no changes.
📁 files — native save dialogs & package output
chooseOutputDir() → string | null
savePackage(files, outputDir)
→ SaveResult
chooseOutputDir() opens a native OS folder picker. In the web build, the existing JSZip download still works unchanged.
🔄 updates — auto-updater events (main → renderer)
onUpdateAvailable(cb) → void
installUpdate() → void
